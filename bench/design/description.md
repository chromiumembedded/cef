These visualizations represent a shift in LLM-agent benchmarking from binary "Success/Failure" metrics toward trajectory-based stochastic analysis. In the context of Mind2Web—a dataset designed for generalist web agents navigating real-world, unfiltered DOM structures—these graphs quantify the entropy of agent reasoning when confronted with anti-bot measures and complex state transitions.1. The Stochastic Mind2Web Pachinko State GridThis visualization maps the probabilistic decay of agent intent. It treats every "Human Step" (the gold standard sequence for a task) as a physical pin in a Galton Board.Mapping Browser Agents (Swift vs. Stable)Agent Stable (Yellow): Represents high-parameter, slower-inference models (e.g., GPT-4o or Claude 3.5 Sonnet). In the grid, these particles exhibit lower horizontal variance. They maintain a tighter Step-Normalized Efficiency ($E$) curve, meaning their path from task start to the "Success Rate (SR)" bin is direct. They are less affected by "Localized page loading latency turbulence," as their reasoning window is broad enough to wait for DOM stability.Agent Swift (Blue): Represents smaller, distilled, or speculative-decoding models. These particles show high "scattering" upon hitting the Anti-Fingerprinting API Access Zone. Because Swift agents often prioritize token throughput over context-window depth, they are more susceptible to "Friction Zones"—recursive DOM loops or hidden honeypot elements—which deflect them toward the Hallucination or Error bins.Benchmark Specifics: The "Friction" ZonesThe grid highlights a specific Mind2Web challenge: Cross-domain generalization.Anti-Fingerprinting API Access Zone: Maps to websites that execute heavy telemetry scripts (e.g., Cloudflare Turnstile, Akamai). The graph shows particles clustering here, representing agents getting stuck in "Are you a robot?" loops.Latency Turbulence: Visualizes the agent's reaction to asynchronous loading. Swift agents often attempt to interact with elements before the DOMContentReady event, leading to the chaotic downward trajectories seen in the center-right of the graph.2. Mind2Web DOM-State Cladogram of "Extinct" EventsThe Cladogram moves from probability to topology. It maps the "evolutionary" path of a task execution, where each bifurcation (split) is a DOM-state transition.The Anatomy of "Extinction"In this context, "Extinction" refers to a terminal state where the agent’s trajectory becomes irrecoverable.Line Thickness (Compute/Token Cost): The "Stable" paths are often thicker, indicating higher token consumption per state transition. The "Swift" paths are thinner but more numerous, showing a "shotgun" approach to navigation that often ends in the FP-Extinction Node (a failure specifically caused by Fingerprinting blocks).The Chaotic Knot (Sudden Localized Page Reloads): This represents a specific failure mode in Mind2Web where an agent’s action (like clicking a malformed button) triggers a full-page refresh, wiping the agent's "memory" of the previous DOM state. In the graph, the lines tangle into a high-entropy knot, representing the agent spinning its wheels (consuming tokens) without advancing the task.Numerical Reasoning Error: Note the red bulbous nodes.
